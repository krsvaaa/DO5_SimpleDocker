# DO5_SimpleDocker

Проект по работе с Docker и создании мини‑веб‑сервера (Docker, Nginx, C, FastCGI, Docker Compose). 

В директории **src** есть файл **REPORT.md**, в котором описаны шаги по работе с Docker.

## Часть 1

В **REPORT.md** показаны загрузка официального образа docker, запуск контейнера, проброс портов для nginx и перезапуск контейнера.


## Часть 2

В **REPORT.md** показана настройка **nginx.conf** для отображения в браузере статуса сервера nginx, работающего внутри контейнера.


## Часть 3 (файлы в src/task3)

В **REPORT.md** показано создание сервера nginx с использованием FastCGI, который будет выводить **Hello World!** при обращении к нему.

Чтобы проверить работу контейнера, вводим:

```bash
./launch.sh
```
Дальше с локальной машины заходим на ```http://localhost:81/```, на странице будет выведено "Hello World!"


## Часть 4 (файлы в src/task4)

В **REPORT.md** показан процесс решения задачи. Здесь, как и в задании 3, создается сервер nginx c использованием FastCGI. Файлы лежат в папке src/part4.

Для проверки работы вводим:

```bash
./launch.sh
```

В браузере по пути ```http://localhost:80/``` выводится "Hello World!", а по пути ```http://localhost:80/status``` выводится статус сервера.


## Часть 5 (файлы в src/task5)

В этом задании была проведена работа по поиску ошибок с помощью Dockle и их исправлению. Порядок действий описан в **REPORT.md**

## Часть 6 (файлы в src/task6)

Эта задача по написанию простого docker compose, который поднимает контейнер из части 5 и контейнер с nginx, который проксирует запросы с 8080 порта на 81 порт первого контейнера. Был произведен проброс портов с 8080 порта второго контейнера на 80 порт локальной машины. Порядок действий описан в **REPORT.md**.

Для проверки работы вводим:

```bash
docker compose up --build
```
После запуска в реальном времени появляются логи контейнеров. В браузере на ```http://localhost:80/``` выводится "Hello World!", а на ```http://localhost:80/status``` выводится статус сервера. В логах контейнеров появляются запросы к ```http://localhost:80/``` и ```http://localhost:80/status```
